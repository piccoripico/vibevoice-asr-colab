# VibeVoice-ASR Colab Notebook

Ang notebook na ito ay nagsasalin ng mga audio o video file.

## Mga Tampok

- **Libreng gamitin:** Ang notebook na ito ay gumagamit ng [VibeVoice-ASR](https://github.com/microsoft/VibeVoice), na open-source ng Microsoft noong Enero 21, 2026, kasama ng Google Colab na libreng GPU runtime. Upang gawing magagamit ang modelo sa mga libreng GPU, ang notebook na ito ay gumagamit ng quantized na VibeVoice-ASR na modelo.
- **Walang nakapirming limitasyon sa laki ng file sa gilid ng notebook:** Maaaring iproseso ang malalaking audio at video file sa pamamagitan ng paghahati sa mga ito sa mga segment.
- **Batch processing:** Maraming audio at video file ang maaaring iproseso sa isang pagtakbo.
- **Google Drive suporta:** Maaaring direktang i-upload ang mga file, o maaaring iproseso ang mga file na audio/video na nakaimbak sa Google Drive.
- **Ganap na online:** Ang mga pag-download at pagproseso ay isinasagawa sa Google Colab. Walang ginagamit na lokal na mapagkukunan ng computing.
- **Maraming format ng output:** Ang notebook ay naglalabas ng orihinal na JSON transcript at nagmula sa TXT, CSV, Markdown, Excel, at SRT na mga subtitle na file.

## Layout ng Imbakan

```text
notebook_template/
  VibeVoice_ASR_Colab.template.ipynb
locales/
  *.json
notebooks/
  VibeVoice_ASR_Colab_<language-code>.ipynb
docs/
  README/
    README.<language-code>.md
scripts/
  build_readmes.py
  build_notebooks.py
  check_notebooks.py
  translate_full_locales.py
```

Ang template notebook ay ang source notebook at naglalaman ng mga placeholder tulad ng
`{{settings.input_mode_help}}`. Ang mga lokal na file ay nagbibigay ng mga halagang tukoy sa wika
para sa Colab-facing copy gaya ng `# @title`, `# @markdown`, mga komento, at mga napili
mga mensahe ng gumagamit. Ang bawat lokal ay naglalaman ng kumpletong hanay ng placeholder mula sa
English source locale.

Naka-commit ang mga nabuong notebook para manatiling stable ang mga link ng Colab.

## Bumuo

```bash
python scripts/build_readmes.py
python scripts/build_notebooks.py
```

## I-refresh ang Mga Pagsasalin

```bash
python scripts/translate_full_locales.py
python scripts/build_readmes.py
python scripts/build_notebooks.py
python scripts/check_notebooks.py
```

`translate_full_locales.py` machine-isinasalin ang buong English locale sa
iba pang mga lokal na file habang pinapanatili ang mga token ng code gaya ng `INPUT_MODE`,
`Google Drive`, `# @markdown`, at mga identifier ng modelo. Suriin ang isinalin na kopya
bago ang isang pormal na pagpapalabas.

## Mga tseke

```bash
python scripts/check_notebooks.py
```

Ang mga pagsusuri sa daloy ng trabaho ng CI na ang mga nabuong notebook ay napapanahon, walang na-save
ang mga cell output, i-parse bilang JSON, ay walang mga hindi naresolbang placeholder, at ang code na iyon
Ang mga cell ay wastong Python syntax.

## Ano ang Masusubok ng CI

Maaaring subukan ng repositoryong ito ang notebook bilang isang naipamahagi na artifact:

- henerasyon mula sa template at mga lokal na file
- validity ng notebook JSON
- walang laman na mga output ng pagpapatupad
- Syntax ng Python sa mga cell ng code
- malinaw na hindi sinasadyang mga lihim
- nakagawa ng mga nabuong notebook na tumutugma sa template

Ang buong transcription path ay nangangailangan ng mga Colab-specific na API, malalaking pag-download ng modelo,
GPU availability, at mga media file na ibinigay ng user. Iyon ay dapat tratuhin bilang isang
manual smoke test o isang hiwalay na GPU/Colab validation workflow.
