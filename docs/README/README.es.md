# VibeVoice-ASR Cuaderno de colaboración

Este cuaderno transcribe archivos de audio o vídeo.

## Características

- **De uso gratuito:** Este cuaderno utiliza [VibeVoice-ASR](https://github.com/microsoft/VibeVoice), cuyo código abierto fue abierto por Microsoft el 21 de enero de 2026, junto con el tiempo de ejecución gratuito GPU de Google Colab. Para que el modelo se pueda utilizar en GPU libres, este portátil utiliza un modelo VibeVoice-ASR cuantificado.
- **No hay límite fijo de tamaño de archivo en el lado del portátil:** Se pueden procesar archivos de audio y vídeo de gran tamaño dividiéndolos en segmentos.
- **Procesamiento por lotes:** Se pueden procesar varios archivos de audio y video en una sola ejecución.
- Soporte **Google Drive:** Los archivos se pueden cargar directamente o se pueden procesar archivos de audio/video almacenados en Google Drive.
- **Completamente en línea:** Las descargas y el procesamiento se realizan en Google Colab. No se utilizan recursos informáticos locales.
- **Múltiples formatos de salida:** El cuaderno genera la transcripción JSON original y los archivos de subtítulos derivados TXT, CSV, Markdown, Excel y SRT.

## Diseño del repositorio

```text
notebook_template/
  VibeVoice_ASR_Colab.template.ipynb
locales/
  *.json
notebooks/
  VibeVoice_ASR_Colab_<language-code>.ipynb
docs/
  README/
    README.<language-code>.md
scripts/
  build_readmes.py
  build_notebooks.py
  check_notebooks.py
  translate_full_locales.py
```

El cuaderno de plantilla es el cuaderno de origen y contiene marcadores de posición como
`{{settings.input_mode_help}}`. Los archivos locales proporcionan valores específicos del idioma
para copias orientadas a Colab como `# @title`, `# @markdown`, comentarios y seleccionados
mensajes de usuario. Cada localidad contiene el conjunto completo de marcadores de posición del
Localidad de origen en inglés.

Los cuadernos generados están comprometidos para que los enlaces de Colab se mantengan estables.

## Construir

```bash
python scripts/build_readmes.py
python scripts/build_notebooks.py
```

## Actualizar traducciones

```bash
python scripts/translate_full_locales.py
python scripts/build_readmes.py
python scripts/build_notebooks.py
python scripts/check_notebooks.py
```

`translate_full_locales.py` traduce automáticamente la configuración regional en inglés completa al
otros archivos locales mientras se conservan tokens de código como `INPUT_MODE`,
`Google Drive`, `# @markdown` e identificadores de modelo. Revisar la copia traducida
antes de una liberación formal.

## cheques

```bash
python scripts/check_notebooks.py
```

El flujo de trabajo de CI comprueba que los cuadernos generados estén actualizados y no se hayan guardado.
salidas de celda, analizadas como JSON, no contienen marcadores de posición sin resolver, y ese código
Las celdas son sintaxis válida de Python.

## Qué puede probar la CI

Este repositorio puede probar el cuaderno como un artefacto distribuible:

- Generación a partir de los archivos de plantilla y configuración regional.
- cuaderno JSON validez
- salidas de ejecución vacías
- Sintaxis de Python en celdas de código
- secretos accidentales obvios
- cuadernos generados comprometidos que coinciden con la plantilla

La ruta de transcripción completa requiere API específicas de Colab, descargas de modelos grandes,
GPU disponibilidad y archivos multimedia proporcionados por el usuario. Eso debería ser tratado como un
prueba de humo manual o un flujo de trabajo de validación GPU/Colab separado.
