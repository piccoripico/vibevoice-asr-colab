# VibeVoice-ASR دفتر كولاب

يقوم هذا الكمبيوتر المحمول بنسخ ملفات الصوت أو الفيديو.

## الميزات

- **مجاني الاستخدام:** يستخدم هذا الكمبيوتر الدفتري [VibeVoice-ASR](https://github.com/microsoft/VibeVoice)، الذي قامت Microsoft بفتح مصدره في 21 يناير 2026، جنبًا إلى جنب مع وقت التشغيل المجاني Google Colab GPU. لجعل النموذج قابلاً للاستخدام على وحدات معالجة الرسومات المجانية، يستخدم هذا الكمبيوتر الدفتري نموذجًا كميًا VibeVoice-ASR.
- **لا يوجد حد ثابت لحجم الملف على جانب الكمبيوتر المحمول:** يمكن معالجة ملفات الصوت والفيديو الكبيرة عن طريق تقسيمها إلى أجزاء.
- **معالجة الدفعات:** يمكن معالجة ملفات الصوت والفيديو المتعددة في عملية تشغيل واحدة.
- **Google Drive الدعم:** يمكن تحميل الملفات مباشرة، أو يمكن معالجة ملفات الصوت/الفيديو المخزنة في Google Drive.
- **متصل بالكامل:** يتم إجراء التنزيلات والمعالجة على Google Colab. لا يتم استخدام أي موارد حوسبة محلية.
- **تنسيقات إخراج متعددة:** يقوم دفتر الملاحظات بإخراج النص الأصلي JSON وملفات الترجمة المشتقة TXT وCSV وMarkdown وExcel وSRT.

## تخطيط المستودع

```text
notebook_template/
  VibeVoice_ASR_Colab.template.ipynb
locales/
  *.json
notebooks/
  VibeVoice_ASR_Colab_<language-code>.ipynb
docs/
  README/
    README.<language-code>.md
scripts/
  build_readmes.py
  build_notebooks.py
  check_notebooks.py
  translate_full_locales.py
```

دفتر ملاحظات القالب هو دفتر الملاحظات المصدر ويحتوي على عناصر نائبة مثل
`{{settings.input_mode_help}}`. توفر الملفات المحلية قيمًا خاصة باللغة
للنسخة المواجهة لـ Colab مثل `# @title` و`# @markdown` والتعليقات والمحددة
رسائل المستخدم. تحتوي كل لغة على العنصر النائب الكامل الذي تم تعيينه من ملف
لغة المصدر الإنجليزية.

يتم الالتزام بدفاتر الملاحظات التي تم إنشاؤها بحيث تظل روابط Colab مستقرة.

## يبني

```bash
python scripts/build_readmes.py
python scripts/build_notebooks.py
```

## تحديث الترجمات

```bash
python scripts/translate_full_locales.py
python scripts/build_readmes.py
python scripts/build_notebooks.py
python scripts/check_notebooks.py
```

`translate_full_locales.py` يترجم الآلة اللغة الإنجليزية الكاملة إلى ملف
ملفات محلية أخرى مع الحفاظ على رموز التعليمات البرمجية مثل `INPUT_MODE`،
`Google Drive`، `# @markdown`، ومعرفات النماذج. مراجعة النسخة المترجمة
قبل الافراج الرسمي.

## الشيكات

```bash
python scripts/check_notebooks.py
```

يتحقق سير عمل CI من أن دفاتر الملاحظات التي تم إنشاؤها محدثة، ولم يتم حفظها
مخرجات الخلية، وتحليلها كـ JSON، لا تحتوي على عناصر نائبة لم يتم حلها، وهذا الرمز
الخلايا هي بناء جملة بايثون صالح.

## ما الذي يمكن لـ CI اختباره

يمكن لهذا المستودع اختبار دفتر الملاحظات باعتباره قطعة أثرية قابلة للتوزيع:

- توليد من القالب والملفات المحلية
- دفتر الملاحظات JSON الصلاحية
- مخرجات التنفيذ الفارغة
- بناء جملة بايثون في خلايا التعليمات البرمجية
- أسرار عرضية واضحة
- دفاتر الملاحظات المولدة المخصصة المطابقة للقالب

يتطلب مسار النسخ الكامل واجهات برمجة التطبيقات الخاصة بـ Colab، وتنزيلات النماذج الكبيرة،
GPU التوفر وملفات الوسائط المقدمة من قبل المستخدم. وينبغي التعامل مع ذلك على أنه أ
اختبار الدخان اليدوي أو سير عمل منفصل للتحقق من صحة GPU/Colab.
