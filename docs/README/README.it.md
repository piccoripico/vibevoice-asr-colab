# VibeVoice-ASR Taccuino Colab

Questo taccuino trascrive file audio o video.

## Caratteristiche

- **Utilizzo gratuito:** questo notebook utilizza [VibeVoice-ASR](https://github.com/microsoft/VibeVoice), reso open source da Microsoft il 21 gennaio 2026, insieme al runtime gratuito GPU di Google Colab. Per rendere il modello utilizzabile su GPU libere, questo notebook utilizza un modello VibeVoice-ASR quantizzato.
- **Nessun limite fisso per le dimensioni dei file sul notebook:** file audio e video di grandi dimensioni possono essere elaborati suddividendoli in segmenti.
- **Elaborazione batch:** è possibile elaborare più file audio e video in un'unica esecuzione.
- Supporto **Google Drive:** i file possono essere caricati direttamente oppure i file audio/video archiviati in Google Drive possono essere elaborati.
- **Completamente online:** I download e l'elaborazione vengono eseguiti su Google Colab. Non vengono utilizzate risorse informatiche locali.
- **Formati di output multipli:** Il notebook genera la trascrizione originale JSON e i file di sottotitoli derivati ​​TXT, CSV, Markdown, Excel e SRT.

## Disposizione del deposito

```text
notebook_template/
  VibeVoice_ASR_Colab.template.ipynb
locales/
  *.json
notebooks/
  VibeVoice_ASR_Colab_<language-code>.ipynb
docs/
  README/
    README.<language-code>.md
scripts/
  build_readmes.py
  build_notebooks.py
  check_notebooks.py
  translate_full_locales.py
```

Il taccuino modello è il taccuino di origine e contiene segnaposto come
`{{settings.input_mode_help}}`. I file locali forniscono valori specifici della lingua
per copie rivolte a Colab come `# @title`, `# @markdown`, commenti e selezionati
messaggi utente. Ogni locale contiene il set completo di segnaposto dal file
Localizzazione di origine inglese.

I taccuini generati vengono impegnati in modo che i collegamenti Colab rimangano stabili.

## Costruire

```bash
python scripts/build_readmes.py
python scripts/build_notebooks.py
```

## Aggiorna traduzioni

```bash
python scripts/translate_full_locales.py
python scripts/build_readmes.py
python scripts/build_notebooks.py
python scripts/check_notebooks.py
```

`translate_full_locales.py` traduce automaticamente la versione inglese completa nel formato
altri file locali preservando i token di codice come `INPUT_MODE`,
`Google Drive`, `# @markdown` e identificatori del modello. Controlla la copia tradotta
prima di un rilascio formale.

## Controlli

```bash
python scripts/check_notebooks.py
```

Il flusso di lavoro CI controlla che i notebook generati siano aggiornati e non siano stati salvati
gli output delle celle, analizzati come JSON, non contengono segnaposto non risolti e quel codice
le celle hanno la sintassi Python valida.

## Cosa può testare l'IC

Questo repository può testare il notebook come artefatto distribuibile:

- generazione dal modello e dai file locali
- taccuino JSON validità
- output di esecuzione vuoti
- Sintassi Python nelle celle di codice
- evidenti segreti accidentali
- taccuini generati impegnati corrispondenti al modello

Il percorso di trascrizione completo richiede API specifiche di Colab, download di modelli di grandi dimensioni,
GPU disponibilità e file multimediali forniti dall'utente. Questo dovrebbe essere trattato come a
test del fumo manuale o un flusso di lavoro di convalida GPU/Colab separato.
